# FacilityPro — Invoices Feature

## Overview
The invoice system handles billing for completed work. Invoices can be created manually or auto-generated from the complaint workflow (when status reaches CLIENT_CONFIRMED). They support line items, taxes, discounts, shipping, and multiple payment tracking fields.

## Invoice Statuses (6)
| Status | Description |
|--------|-------------|
| `DRAFT` | Initial draft (auto-created by workflow) |
| `PENDING` | Pending approval |
| `APPROVED` | Approved, ready to send |
| `PAID` | Payment received |
| `CANCELLED` | Cancelled |
| `OVERDUE` | Past due date, unpaid |

## Invoice Number Format
`INV/SMSB/01/{YEAR}/{SEQ}` — generated by `generateInvoiceNumber()` in `src/lib/auth.ts`.

## Invoice in Workflow
In the complaint workflow:
1. `CLIENT_CONFIRMED` → system auto-creates draft invoice (status: `DRAFT_INVOICE`)
2. Finance/admin reviews and approves (`INVOICE_APPROVED`)
3. Finance/admin sends to customer (`INVOICE_SENT`, requires `sentVia`)
4. Customer pays (`PAID`, requires `paymentMethod`, `paymentRef`, `paidAt`)
5. Complaint closes automatically
